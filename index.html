Summ-lib Documentation

How to use summ-lib in your project:

  1. Include "<script src="http://simon-gitsumm.github.io/summ/summ-lib.js"></script>  in you html document after 'phaser.js' but before your games .js file
  2. Download http://simon-gitsumm.github.io/summ/summ-lib.d.ts and place in your project folder. Include it as a Typescript refernce to your typescript files using /// <reference path="summ-lib.d.ts" />

summ.UserDataMessages

static setUserData(data: Object, callback?: Function, callbackContext?: Object): void;

Saves per player, per game data to the server. (Replaces existing data)

  • data - An object containing the data you wish to save. It is best to save all data as strings as they will be parsed and returned as strings when using getUserData.
  • callback (optional) - a function which will be called when a reply from the server is recieved. Its first argument will contain the field succeed of type boolean which will tell whether or not the data was saved
  • callbackContext(required if using callback)- the context for the callback function

static getUserData(callback: Function, callbackContext: Object): void;

Loads current players data for the current game.

  • callback - a function which will be called when the data is recieved. It's first argument will contain an object with the field data of type Object containing all the fields that were saved using setUserData. Note: All field in data will be of type String so make sure to parse them as required.
  • callbackContext - a function used to load your assets. It is called from the context specified, usually the game instance.

summ.Preloader

static load(game: Phaser.Game, loadAssets: Function, context: Object, nextState: string): void;

Loads your assets displaying the gitsum loading bar and logo before loading the next state. Will only work from a webserver as most browsers will block any requests for cross origin files.

  • game - the instance of Phaser.Game to load the images into.
  • loadAssets - a function used to load your assets. It is called from the context specified, usually the game instance.
  • context - the object to be used as the context for loading the assets
  • nextState - a string specifying the new state to change to once the loading is finished

static load(game: Phaser.Game, loadAssets: Function, context: Object, nextState: string): void;

A version of load which works on a local machine. Loads your assets using 'logo.png' and 'loader.png' which should be placed in the projects /assets folder.

  • game - the instance of Phaser.Game to load the images into.
  • loadAssets - a function used to load your assets. It is called from the context specified, usually the game instance.
  • context - the object to be used as the context for loading the assets
  • nextState - a string specifying the new state to change to once the loading is finished

summ.PauseMenu

This object is used to create pause menus for games. It can also be used for generic menus.

constructor(game: Phaser.Game, defaultSprite?: string, defaultOverFrame?: number, defaultOutFrame?: number, defaultDownFrame?: number, defaultUpFrame?: number, defaultScaleX?: number, defaultScaleY?: number, defaultTextStyle?: any, backgroundSprite?: Phaser.Sprite, stretchBackground?: boolean, menuBounds?: Phaser.Rectangle, spreadAlongY?: boolean);

  • game - the instance of Phaser.Game to load the images into.
  • defaultSprite- Specifiy the default sprite to use for all buttons. This should be a string pointing to the asset in phasers game cache
  • defaultOverFrame - Specifiy the default frame to use when the mouse is over the button
  • defaultOutFrame - Specifiy the default frame to use when the mouse is not over the button
  • defaultDownFrame - Specifiy the default frame to use when the button is pressed down
  • defaultUpFrame - Specifiy the default frame to use when the button is released
  • defaultScaleX - Specifiy the default value for the sprites x scale
  • defaultScaleY - Specifiy the default value for the sprites y scale
  • defaultTextStyle - Specifiy the default text style for all the buttons
  • backgroundSprite - Specifiy a sprite that will be placed behind the buttons.This should be a string pointing to the asset in phasers game cache
  • stretchBackground - If true, the background sprite will stretch to fit the menuBounds
  • menuBounds - A rectangle defining the square in which the menu will be placed. The buttons will be centered in this rect.
  • spreadAlongY - If true, the buttons will be spaced evenly along the entire height of the menuBounds. If false, they will group towards the middle.

 

Important Properties:

  • phaserPause: boolean - If true, the pause menu will use Phasers game.paused property to pause the game, otherwise it will use my hacky method which will still allow input.

  • scaleX - The value for the sprites x scale
